## A note on internal references

ADRs are **historical records** — they capture a decision as it stood when it was
made, and are not rewritten afterwards. Some therefore cite internal **RFCs**
(`RFC-NNN`), design documents, earlier internal version numbers, or earlier file paths and module structures that
are part of the project's private development history and are **not included in
this public repository**. Those references are kept for historical accuracy; the
ADR text itself is self-contained and does not depend on them being present.
