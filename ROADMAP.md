# Roadmap

Teaching Suite is at **v0.5.0** — a pre-1.0 release. The tool architecture is
stable and tested (642 tests, 24 MCP tools), and the core reflection cycle
(plan → teach → reflect → improve) works end-to-end. The version reflects honest
maturity: the **pedagogical methodology is still in active development**, and the
tool has not yet been validated with teachers other than the maintainer.

This roadmap describes direction, not dated commitments.

## Now — v0.5.0

- 24 MCP tools across capture, reflection, planning, methodology, and source tracking.
- Methodology v3.0: three nested cycles — *lesson*, *course*, *profession* —
  grounded in named pedagogical frameworks (see [`methodology/`](methodology/)).
- Workspace-locked, no network access; bilingual (code English, methodology and
  UX Swedish).

## Towards 1.0

1.0 means the methodology and the tool have been **validated in real use beyond
the maintainer**, and the core pedagogical principles are stable.

- **External pilot** — a small pilot with external teachers (planned for autumn
  2026) to test the methodology and workflows in real classrooms.
- **Methodology maturation** — the core principles and the cycle documents
  continue to be developed and refined. This is the largest body of remaining
  work, and the main reason the project is pre-1.0.
- **Stability** — the tool surface and the workspace structure settle as the
  methodology stabilises.

## Not planned for 1.0

- Automation of pedagogical judgement — the design principle is *scaffolding,
  not automation*: the teacher thinks, the tool structures.
- Handling of student personal data inside this tool — student data stays in the
  assessment workflow; only insights about teaching flow back.

## Feedback

This is an early, openly-developed project. Issues and ideas are welcome (see
[CONTRIBUTING.md](CONTRIBUTING.md)) — especially from teachers trying it in
practice.
